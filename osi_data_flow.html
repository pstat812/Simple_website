The session layer is responsible for setting up and maintaining the communication linkages between two devices in a network during data transmissions.

Segments
The transport layer adds a Transport Header (TH) to data to form a datagram. The TH contains the transportation information such as the protocol that the user using. A Transport Header using TCP protocol includes source port number, destination port number, sequence number, acknowledgement number, header length, flags, windows size, TCP checksum, urgent pointer and more. The length of Transport header usually ranges from 20 to 60 bytes. If it is using UDP protocol, it only contains source port number, destination port number, header length and checksum of header. The header of UDP datagram have 8 bits only.

Packets
The network determines the routing and forwarding of data. It adds a Network Header (NH) to a datagram to form a packet. A Network Header contains network address information. The length of the header of an IP packet also ranges from 20 to 60 bytes.

Frames
The data link layer is responsible for the network addressing as well as error detection and correction. In this layer, a header and a trailer are added to a packet to form a frame. The Data Link Header (DLH) contains the physical address and the method of error detection and correction. A Data Link Trailer (DLT) is a string of bits that indicates the end of the packet. Ethernet, Wi-Fi and 4G are examples of data link protocols.

Bits
The physical layer transfers frames physically on a LAN. It is responsible for managing the interaction between communication devices in a computer and a network medium.
